# Роли в боте

Теперь роли можно добавлять прямо через Telegram.

## Добавить роль

Формат:

```text
/addrole role_key chat_id Имя | алиас1,алиас2
```

Пример:

```text
/addrole assistant3 123456789 Саид | саид,ассистент,said
```

Что значит:

- `role_key` — короткий ключ роли
- `chat_id` — id сотрудника
- `Имя` — как показывать человека в задачах
- `алиасы` — слова, по которым бот его понимает

## Посмотреть роли

```text
/roles
```

## Удалить роль

```text
/delrole assistant3
```

Удаляются только роли, которые добавлены через `/addrole`.

## Как получить chat id сотрудника

Пусть сотрудник напишет:

```text
/start
/myid
```

После этого можно сразу добавлять его роль.

## Пример рабочего сценария

1. Абдулло пишет `/start`
2. Абдулло пишет `/myid`
3. ты получаешь его `chat_id`
4. добавляешь:

```text
/addrole assistant3 123456789 Абдулло | абдулло,ассистент
```

5. после этого можно писать:

```text
Завтра съемка в 12, Абдулло
Сегодня в 18 публикация, assistant3
```

## Где это хранится

Базовые роли лежат в `config.example.json`.

Роли, добавленные через Telegram, хранятся отдельно в:

```text
/data/custom_roles.json
```

Этот файл создаёт сам бот.
