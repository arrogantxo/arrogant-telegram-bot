# Telegram Reminder Bot

Удобный Telegram-бот для команды. Он помогает ставить задачи простыми сообщениями, напоминает о съемках, монтаже, публикациях и позволяет сотрудникам смотреть свои задачи без длинных команд.

## Как теперь пользоваться

Главная идея: не нужно помнить длинные команды. Можно писать боту обычным языком.

Примеры:

```text
Завтра съемка в 12, монтажер и видеограф
Сегодня в 18:00 публикация, publisher
Завтра в 10 сценарий для сценариста
Напомни всей команде о съемке завтра в 9
Какие задачи на сегодня?
Мои задачи
```

## Что умеет бот

- принимать обычные текстовые сообщения и превращать их в задачи
- отправлять личные напоминания сотрудникам
- отправлять общее напоминание в командный чат
- показывать задачи на сегодня
- показывать список задач
- принимать голосовые сообщения
- распознавать голос в текст через OpenAI
- понимать простые запросы сотрудников по задачам

## Кто что может

### Админ

Админ может:

- создавать задачи
- создавать общие напоминания всей команде
- смотреть все активные задачи
- отмечать задачи выполненными
- удалять задачи

### Сотрудник

Сотрудник может:

- написать `/start`
- узнать свой `chat_id` через `/myid`
- посмотреть свои задачи через `Мои задачи`
- спросить `Какие задачи на сегодня?`

Сотрудник не может создавать задачи, если его `chat_id` не добавлен в админы.

## Примеры для вашей команды

### Съемка

```text
Завтра съемка в 12, монтажер и видеограф
Сегодня съемка в 16:30, manager и videograf
Напомни всей команде о съемке завтра в 9
```

### Монтаж

```text
Завтра монтаж reels в 15, монтажер
Сегодня в 20 смонтировать reels для скидок, montaj
```

### Публикация

```text
Сегодня в 18 публикация поста, publisher
Завтра в 19 выложить reels, publisher
```

### Сценарии

```text
Завтра в 10 сценарий для сценариста
Сегодня в 14 написать сценарий на reels, scriptwriter
```

### Проверка задач

```text
Какие задачи на сегодня?
Мои задачи
```

## Короткие команды

Бот всё ещё поддерживает команды:

- `/start`
- `/help`
- `/myid`
- `/team`
- `/categories`
- `/users`
- `/list`
- `/today`
- `/done TASK_ID`
- `/delete TASK_ID`
- `/add YYYY-MM-DD HH:MM employee_key category text`
- `/broadcast YYYY-MM-DD HH:MM category text`

Но теперь их можно использовать как запасной вариант, а не как основной режим.

## Голосовые сообщения

Голосом можно:

- поставить задачу
- спросить задачи на сегодня
- попросить напомнить всей команде о съемке

Примеры голосовых:

- "Завтра съемка в 12, монтажер и видеограф"
- "Какие у меня задачи на сегодня?"
- "Напомни всей команде о съемке завтра в 9"

Важно:

- для голосовых нужен `OPENAI_API_KEY`
- если у OpenAI нет квоты или биллинга, голосовые не будут распознаваться

## Как добавить роли

Роли лежат в файле `config.example.json`, секция `team`.

Пример:

```json
"designer": {
  "name": "Designer",
  "chat_id": 444444444
}
```

Что нужно сделать:

1. сотрудник пишет боту `/start`
2. сотрудник пишет `/myid`
3. ты берешь его `chat_id`
4. добавляешь новую роль в `team`
5. при необходимости добавляешь слова для этой роли в `ROLE_ALIASES` внутри `bot.py`

Отдельная короткая инструкция есть в [ROLES.md](/D:/SMM%20Arrogant/%D0%9E%D0%B7%D0%B1%D0%B5%D0%B3%D0%B8%D0%BC%20%D0%BC%D0%BE%D0%BB%D0%BB/04_%D0%9F%D1%80%D0%BE%D0%B5%D0%BA%D1%82%D1%8B/telegram_reminder_bot/ROLES.md).

## Railway Variables

В Railway Variables должны быть:

```text
BOT_TOKEN=...
BOT_DB_PATH=/data/bot_data.db
BOT_USERS_PATH=/data/known_users.json
OPENAI_API_KEY=sk-...
```

## Volume

У Railway volume должен быть подключен к пути:

```text
/data
```

## Если бот не отвечает

Проверь по порядку:

1. Railway сервис должен быть `Active`
2. в Railway logs должен быть `Telegram Reminder Bot started`
3. `BOT_TOKEN` должен быть правильным
4. volume должен быть подключен в `/data`
5. сотрудник должен сначала написать боту `/start`
6. для голосовых нужен рабочий `OPENAI_API_KEY`
