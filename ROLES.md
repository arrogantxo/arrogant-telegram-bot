# Роли в боте

Этот файл нужен, чтобы быстро добавлять новых людей в команду бота.

## Где хранятся роли

Роли лежат в файле `config.example.json`, секция:

```json
"team": {
  "manager": {
    "name": "Arrogant Studios",
    "chat_id": 406716530
  }
}
```

## Как добавить нового сотрудника

### Шаг 1

Попроси сотрудника написать боту:

```text
/start
/myid
```

### Шаг 2

Возьми его `chat_id`.

### Шаг 3

Добавь его в `config.example.json`.

Пример:

```json
"videograf": {
  "name": "Videograf",
  "chat_id": 123456789
}
```

### Шаг 4

Если хочешь, чтобы бот понимал роль по словам вроде `видеограф`, `оператор`, `съемщик`, добавь синонимы в `ROLE_ALIASES` внутри `bot.py`.

Пример:

```python
"videograf": ["videograf", "видеограф", "оператор", "съемщик", "съёмщик"]
```

## Пример добавления новой роли

Допустим, ты хочешь добавить таргетолога.

### В `config.example.json`

```json
"targetolog": {
  "name": "Targetolog",
  "chat_id": 777777777
}
```

### В `bot.py`

В словарь `ROLE_ALIASES`:

```python
"targetolog": ["targetolog", "таргетолог", "реклама"]
```

После этого можно будет писать так:

```text
Завтра в 11 реклама для таргетолога
```

## Если одна роль = один человек с другой задачей

Можно оставить одного и того же человека в нескольких ролях.

Например:

- `montaj`
- `publisher`

оба могут вести на один и тот же `chat_id`, если это один сотрудник.

## После изменения ролей

1. сохрани изменения
2. загрузи файлы в GitHub
3. сделай `Redeploy` в Railway

После этого новые роли начнут работать.
