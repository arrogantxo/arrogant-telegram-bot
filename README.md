# Telegram Reminder Bot

Общий Telegram-бот для команды. Он напоминает о рабочих задачах и дедлайнах.

## Для чего нужен

- напоминания о съемках
- напоминания о сценариях
- напоминания о монтаже
- напоминания о публикациях
- напоминания о созвонах
- напоминания о согласованиях и выездах
- голосовые команды для быстрой постановки задач

## Что умеет

- отправлять личные напоминания сотрудникам
- отправлять напоминания в общий чат
- хранить задачи в SQLite
- показывать список активных задач
- показывать задачи на сегодня
- отмечать задачи выполненными
- удалять задачи
- показывать список сотрудников
- показывать список категорий
- принимать голосовые сообщения
- распознавать голос в текст через OpenAI
- отвечать на голосовые запросы по задачам

## Команды

- `/start` - приветствие
- `/help` - список команд
- `/myid` - показать chat id
- `/team` - список сотрудников из конфига
- `/categories` - список категорий задач
- `/users` - пользователи, которые уже писали боту
- `/add YYYY-MM-DD HH:MM employee_key category text`
- `/broadcast YYYY-MM-DD HH:MM category text`
- `/list`
- `/today`
- `/done TASK_ID`
- `/delete TASK_ID`

## Примеры текстовых команд

```text
/add 2026-04-12 10:00 manager съемка Съемка в Озбегим молл
/add 2026-04-12 14:00 montaj монтаж Смонтировать reels для акции
/add 2026-04-12 18:00 publisher публикация Выложить reels и пост в Instagram
/broadcast 2026-04-12 09:00 съемка Общая съемка для всей команды
```

## Голосовые команды

Бот может принимать голосовые сообщения и распознавать их.

Что можно делать голосом:

- ставить задачу
- спрашивать список задач на сегодня
- просить напомнить всей команде о съемке

Примеры:

- "Завтра в 10 утра напомни монтажеру смонтировать reels"
- "Какие у меня задачи на сегодня?"
- "Напомни всей команде о съемке завтра в 9 утра"

## Настройка Railway

В Railway Variables должны быть:

```text
BOT_TOKEN=...
BOT_DB_PATH=/data/bot_data.db
BOT_USERS_PATH=/data/known_users.json
OPENAI_API_KEY=sk-...
```

## Что означает каждая переменная

- `BOT_TOKEN` — токен Telegram-бота из BotFather
- `BOT_DB_PATH` — путь к SQLite базе задач
- `BOT_USERS_PATH` — путь к файлу известных пользователей
- `OPENAI_API_KEY` — ключ OpenAI для распознавания голосовых сообщений

## Volume

У Railway volume должен быть подключен к пути:

```text
/data
```

## Как узнать chat id

Каждый сотрудник пишет боту:

```text
/start
/myid
```

После этого бот покажет его `chat_id`.

Если не хочется собирать `chat_id` вручную:

- сотрудники просто пишут боту `/start`
- бот сохраняет их в `known_users.json`
- потом можно посмотреть их через `/users`

## Важно

- Бот работает 24/7 на Railway
- Чтобы бот писал сотруднику в личку, сотрудник должен хотя бы один раз написать боту `/start`
- OpenAI API key и Telegram token нельзя хранить в GitHub
- Все секреты нужно хранить только в Railway Variables
